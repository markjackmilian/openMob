## Executive Summary

This feature introduces a "New Session" button in the lateral drawer, positioned above the existing session list. When tapped, the button creates a new empty session on the currently active project (auto-selecting the first available project if none is active) and navigates the user directly to the `ChatPage` ready to start chatting. The session title starts as a placeholder and is updated automatically after the first message via opencode's `title` subagent, with the update reflected in real time in both the drawer and the chat header.

---

## Scope

### In Scope
- "New Session" button in the lateral drawer, positioned above the session list
- Auto-selection of the first available project from the server when no project is currently active
- Session creation via opencode API (`POST /session` with active `projectId`)
- Automatic navigation to `ChatPage` with the newly created session loaded and ready for input
- Session created with an empty/placeholder title — no user input required
- Real-time title update after the first message: openMob listens to the SSE event carrying the title generated by opencode's `title` subagent and updates the session title in the drawer list and `ChatPage` header without a page reload
- Loading state on the button during creation (disabled + visual indicator) to prevent duplicate taps
- Inline error message in the drawer on creation failure
- Drawer session list updated immediately after creation (new session appears at the top)
- `NewSessionCommand` on `DrawerViewModel` (or equivalent drawer ViewModel)
- New `CreateSessionAsync(string projectId)` method on `ISessionService`

### Out of Scope
- Manual title input by the user
- Session creation when no server is connected/reachable
- Server-side `title` subagent logic (handled entirely by opencode)
- Any other changes to the drawer layout, styles, or navigation items beyond the new button
- Deletion or renaming of sessions from the drawer (separate feature)

## Functional Requirements

> Requirements are numbered for traceability.

1. **[REQ-001]** The lateral drawer must display a "New Session" button above the session list. The button is always visible regardless of whether a project is active.

2. **[REQ-002]** When the user taps "New Session" and a project is currently active, the session is created on that project without any additional user interaction.

3. **[REQ-003]** When the user taps "New Session" and no project is currently active, the app must:
   a. Fetch the list of available projects from the server.
   b. Set the first project in the returned list as the active project.
   c. Proceed with session creation on that project.
   If no projects are available, display an inline error message and abort (no navigation).

4. **[REQ-004]** Session creation is performed by calling the opencode API (`POST /session`) with the active `projectId`. The API response must return the new `sessionId`.

5. **[REQ-005]** The new session is created with an empty or placeholder title (e.g., `"New Session"` or empty string). No title input is requested from the user at creation time.

6. **[REQ-006]** Immediately after successful creation, the app closes the drawer and navigates to `ChatPage` with the new session loaded. The chat input field must be focused and ready for the user's first message.

7. **[REQ-007]** The new session must appear at the top of the session list in the drawer immediately after creation, before any title is assigned.

8. **[REQ-008]** While the creation operation is in progress (from tap to successful navigation), the "New Session" button must be disabled and display a loading indicator to prevent duplicate session creation.

9. **[REQ-009]** If the creation API call fails (network error, server error, no projects available), the button returns to its normal state and an inline error message is displayed within the drawer. No navigation occurs.

10. **[REQ-010]** After the user sends the first message in the new session, opencode's `title` subagent generates a title server-side and emits an SSE event. openMob must handle this SSE event and update the session title in:
    - The drawer session list item for that session.
    - The `ChatPage` header title.
    Both updates must happen without a full page reload or manual refresh.

11. **[REQ-011]** The `DrawerViewModel` (or equivalent) must expose:
    - `NewSessionCommand` (`[AsyncRelayCommand]`) — triggers the full creation flow.
    - `IsCreatingSession` (`bool`, `[ObservableProperty]`) — true while the API call is in progress; drives button disabled state and loading indicator.
    - `CreationError` (`string?`, `[ObservableProperty]`) — set on failure, cleared at the start of each new attempt.

12. **[REQ-012]** `ISessionService` must be extended with a new method:
    ```
    Task<SessionDto> CreateSessionAsync(string projectId, CancellationToken ct = default)
    ```
    The method calls `POST /session`, deserialises the response into a `SessionDto`, and returns it. On non-success HTTP status, it throws an exception with a user-readable message.

## Acceptance Criteria

> Each criterion maps to one or more functional requirements.

- [ ] **[AC-001]** Given an active project, when the user taps "New Session" in the drawer, then a new session is created and `ChatPage` opens with that session within 3 seconds. *(REQ-002, REQ-004, REQ-006)*

- [ ] **[AC-002]** Given no active project and at least one project available on the server, when the user taps "New Session", then the first project is automatically set as active, a session is created on it, and `ChatPage` opens. *(REQ-003, REQ-004, REQ-006)*

- [ ] **[AC-003]** Given no active project and no projects available on the server, when the user taps "New Session", then an inline error message is shown in the drawer and no navigation occurs. *(REQ-003, REQ-009)*

- [ ] **[AC-004]** While session creation is in progress, the "New Session" button is disabled and shows a loading indicator; tapping it again has no effect. *(REQ-008, REQ-011)*

- [ ] **[AC-005]** Given a creation API failure, when the error occurs, then the button returns to its normal enabled state and an inline error message is visible in the drawer. *(REQ-009, REQ-011)*

- [ ] **[AC-006]** The new session appears at the top of the drawer session list immediately after creation, before any title is assigned. *(REQ-007)*

- [ ] **[AC-007]** After the user sends the first message in the new session, the session title in the drawer list and in the `ChatPage` header updates automatically to the title generated by the `title` subagent, without a page reload. *(REQ-010)*

- [ ] **[AC-008]** `ISessionService.CreateSessionAsync` throws a descriptive exception on non-success HTTP response; the exception message is surfaced as `CreationError` in the drawer. *(REQ-012, REQ-009)*

## Notes for Technical Analysis

> This section is addressed to the agent that will perform the technical implementation analysis.

- **opencode API — session creation:** Confirm the exact endpoint (`POST /session`?), required headers, request body schema (does it require `projectId`? any other fields?), and the response DTO shape. Map the response to a `SessionDto` (check if one already exists or needs to be created/extended).

- **opencode SSE — title update event:** The `title` subagent is a known system agent (`Hidden=true`, confirmed in `spec-session-context-sheet-2of3-agent-model`). After the first user message, opencode invokes it server-side and emits an SSE event with the generated title. Identify the SSE event type name and payload. Determine whether `OpencodeApiClient` already parses this event type or if a new case must be added.

- **Active project resolution:** Identify the existing service/repository responsible for tracking the active project (likely `IServerConnectionRepository` or a dedicated `IProjectService`). Determine whether a `GetActiveProjectIdAsync()` method already exists and whether a `SetActiveProjectAsync(string projectId)` or equivalent is available. If auto-selection requires fetching the project list, confirm the existing `GET /project` (or equivalent) API call.

- **DrawerViewModel location:** Identify the existing ViewModel that backs the lateral drawer (name, file path, current responsibilities). Determine whether it already holds the session list and how it is currently populated.

- **Session title in ChatViewModel:** Check whether `ChatViewModel` already exposes a `SessionTitle` (or equivalent) observable property bound to the `ChatPage` header. If not, it must be introduced. The SSE title-update handler must dispatch to the UI thread (established pattern — see `spec-session-context-sheet-1of3-core` and `adr-weakreferencemessenger-viewmodel-communication`).

- **WeakReferenceMessenger vs direct navigation:** After `NewSessionCommand` creates the session, it must navigate to `ChatPage` and pass the new `sessionId`. Determine whether the existing navigation pattern uses route parameters (Shell navigation with query parameters) or a messenger-based approach. Use the established `INavigationService` abstraction.

- **Established patterns to respect:**
  - `[AsyncRelayCommand]` + `IsCreatingSession` busy flag (CommunityToolkit.Mvvm).
  - `_ = Task` discard pattern for fire-and-forget saves (do not use `async void`).
  - `ConfigureAwait(false)` in all Core service/repository calls.
  - `SentryHelper.CaptureException(ex, extras)` on all caught exceptions.
  - `LoadError` / `CreationError` string property pattern for inline error display (established in `bugfix/server-delete-navigation`, 2026-03-19).
  - UI thread dispatch via `_dispatcher.Dispatch(...)` for observable property mutations triggered by background events (established in `spec-session-context-sheet-1of3-core`).

- **Related files to investigate:**
  - `src/openMob.Core/ViewModels/` — locate DrawerViewModel (or equivalent)
  - `src/openMob.Core/Services/ISessionService.cs` and `SessionService.cs`
  - `src/openMob.Core/Infrastructure/Http/` — `OpencodeApiClient.cs` for SSE event parsing
  - `src/openMob.Core/Infrastructure/Http/Dtos/Opencode/` — existing DTOs
  - `src/openMob/Views/` — drawer XAML and `ChatPage.xaml`
  - `src/openMob.Core/ViewModels/ChatViewModel.cs` — session title property and SSE handling
